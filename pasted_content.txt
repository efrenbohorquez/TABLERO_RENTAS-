### MÓDULO 1: CICLO DE VIDA DEL DATO Y ARQUITECTURA ETL (DATA ENGINEERING)
Para garantizar que el sistema compare el pronóstico con la realidad, debes implementar un pipeline de ingesta continua.
1. **Ingesta Dinámica:** Crea clases en Python orientadas a objetos (OOP) para automatizar la extracción y carga (ETL) de nuevos datos transaccionales de recaudo.
2. **Preprocesamiento Automatizado:** Implementa funciones de limpieza que manejen valores nulos y generen las variables sintéticas críticas: `lag_12` (rezago anual), medias móviles (3 y 6 meses) y variables *dummy* para el calendario de festividades municipales.
3. **Base de Datos Analítica:** Estructura un modelo de datos (ej. esquema de estrella) que almacene `y_real`, `y_pred_xgboost`, `y_pred_lstm` y `y_pred_ensemble` indexado por `DIVIPOLA` (municipio) y `fecha`.

### MÓDULO 2: MOTOR PREDICTIVO MULTI-HORIZONTE (XGBOOST + LSTM)
El núcleo del aplicativo debe soportar granularidad múltiple para distintos perfiles de usuario.
1. **Agregación Temporal:** El código debe permitir re-muestrear (*resample*) la serie de tiempo desde datos diarios hacia agregaciones semanales, mensuales y bimensuales (para suavizar el ruido administrativo).
2. **Entrenamiento y Predicción:** 
   * **XGBoost:** Configúralo para capturar relaciones no lineales y el impacto de variables exógenas (PIB, IPM, Festividades).
   * **LSTM:** Diseña la arquitectura de la red neuronal (TensorFlow/Keras) para capturar secuencias complejas y dependencias a largo plazo.
3. **Mecanismo de Monitoreo (Model Drift):** Crea una función que evalúe diariamente si la divergencia entre el recaudo real ingresado y el pronosticado supera un umbral estadístico, activando un reentrenamiento automatizado.

### MÓDULO 3: INTERFAZ EJECUTIVA Y VISUALIZACIÓN INTERACTIVA
El usuario final es un gerente; la interfaz (usa Streamlit o Dash) debe ser intuitiva, prescindiendo de código y enfocándose en la "Gestión Predictiva Territorial".
Debe contener:
1. **Selector de Contexto:** Filtro por Departamento, Municipio y Tipología Municipal (A: Consolidado, B: Emergente, C: Dependiente, D: Crítico).
2. **Gráfico Interactivo Principal:** Una serie de tiempo dinámica (usando Plotly) que superponga el *Recaudo Real* (línea sólida) vs. *Recaudo Pronosticado* (línea punteada) con sus intervalos de confianza del 95% (bandas sombreadas).

### MÓDULO 4: LOS 3 KPIs CRÍTICOS PARA GERENCIA
El dashboard debe calcular y mostrar en tarjetas destacadas los siguientes 3 indicadores, actualizándose con cada nueva ingesta de datos:
* **KPI 1: Índice de Eficiencia Predictiva (IEP):** Fórmula: `((Recaudo_Real - Recaudo_Pronosticado) / Recaudo_Pronosticado) * 100`. Define el estado del municipio (ej. un IEP negativo sostenido indica ineficiencia o déficit).
* **KPI 2: MAPE Global y Local (Salud del Modelo):** Error Porcentual Absoluto Medio del último horizonte evaluado. Si el MAPE > 15%, debe emitir una advertencia de "Alta Incertidumbre".
* **KPI 3: Semáforo de Riesgo Fiscal (Brecha de Recaudo):** 
    * 🔴 **Rojo:** Recaudo Real cae por debajo del límite inferior del intervalo de confianza.
    * 🟡 **Amarillo:** Recaudo Real < Pronóstico, pero dentro del intervalo.
    * 🟢 **Verde:** Recaudo Real >= Pronóstico.
